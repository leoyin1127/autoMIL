<objective>
Wire the cell layer into the submit pathway. Add `--budget-seconds` / `--safety-buffer-seconds` Click options to `automil submit`, call `get_or_create_cell()` before writing the queue spec, raise `ClickException` if the cell is refusing new submits, and stamp `spec.metadata.cell_id` on every successful submit. Extend `templates/config.yaml.j2` with the consumer-facing `cap:` and `training:` sections.

Purpose: D-116, D-117, D-134. This is the agent-facing edge of the cap layer — when a cell is full, submit fails fast with a structured error telling the operator what to do (Pitfall-9 defence). `metadata.cell_id` is the cap-membership tag the daemon's `_running_in_cell` (Plan 04-08) reads to count in-cell experiments.

Output: 1 modified module (submit.py), 1 modified template (config.yaml.j2), 1 new test file. No daemon work here — that is Plan 04-08.
</objective>
