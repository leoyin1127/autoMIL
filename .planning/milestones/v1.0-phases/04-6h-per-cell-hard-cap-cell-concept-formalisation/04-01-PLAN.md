```python
def _atomic_write_text(path: Path, content: str) -> None:
    path.parent.mkdir(parents=True, exist_ok=True)
    tmp_fd, tmp_path = tempfile.mkstemp(dir=str(path.parent), suffix=".tmp")
    try:
        with os.fdopen(tmp_fd, "w") as f:
            f.write(content)
        os.replace(tmp_path, str(path))
    except Exception:
        try:
            os.unlink(tmp_path)
        except OSError:
            pass
        raise
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Daemon process → cells/<id>.json on disk | Daemon is the sole writer; CLI reads only. |
| time.time() oracle | Used as the canonical wall-clock; spoofing is the sandbagging vector. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | started_at field in cells/<id>.json | mitigate | started_at written ONCE at cell creation by registry.get_or_create_cell (Plan 04-06); Cell is frozen so no in-place mutation; write_cell goes through atomic mkstemp + os.replace so partial writes are impossible. Subsequent writes only update status field, leaving started_at untouched. |
| T-04-02 | Tampering | consumed_seconds clock | mitigate | D-111 computed model: consumed_seconds = time.time() - started_at on every call; no accumulator state can be drifted by a missed tick. Daemon-restart test (Plan 04-11) asserts this invariant. |
| T-04-03 | Tampering | atomic write race (cross-filesystem) | mitigate | tempfile.mkstemp(dir=str(cells_dir)) keeps tmp file on same filesystem as destination → os.replace is atomic on POSIX (Pitfall 2). Acceptance grep verifies `dir=str(cells_dir)` is present. |
</threat_model>
