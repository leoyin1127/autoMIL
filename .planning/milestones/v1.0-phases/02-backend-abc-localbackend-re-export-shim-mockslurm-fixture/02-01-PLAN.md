# Plan 02-01: Backend ABC + Dataclasses + Test Package Skeleton

## Objective

Ship the type system foundation for Phase 2: the `Backend` ABC with its five abstract methods, three frozen dataclasses (`JobHandle`, `JobSpec`, `JobState`), the `backends` package `__init__.py`, and the `tests/backends/` package scaffold with the `wait_for_state` polling helper. Plans 02-04 through 02-08 all build against these contracts — this plan must land before any backend implementations.

Per D-51, the `backends/` package is created from scratch; per D-52–D-54, the three dataclasses + enum are frozen/str-Enum for JSON-safety and hashability; per D-55–D-58, the five method docstrings encode the fire-and-forget / snapshot / eventually-consistent contract the contract test will verify.

Purpose: No implementation exists without the ABC to subclass. If `base.py` is wrong, LocalBackend and MockSLURMBackend will freeze wrong semantics.

Output: 4 new files. 387 existing tests unchanged. `from automil.backends import Backend, JobHandle, JobSpec, JobState` works.

## Threat Model (STRIDE + disposition)

| ID | Threat | STRIDE | Severity | Disposition |
|----|--------|--------|----------|-------------|
| T-02-01-S01 | `JobSpec.env` tuple contains secrets (API keys) passed to subprocess | Information Disclosure | MEDIUM | Mitigate: `env` is a `tuple[tuple[str,str],...]` explicitly whitelisted by the caller (CLN-02 already enforces whitelist at submit time); `JobSpec` frozen prevents post-construction injection; no secret persistence in Phase 2 |
| T-02-01-S02 | `JobHandle.opaque_id` collision — two jobs get the same opaque_id | Spoofing | LOW | Mitigate: LocalBackend uses PID (unique per process), MockSLURM uses monotonic counter; contract test `test_opaque_id_unique` enforces uniqueness |
| T-02-01-S03 | `JobSpec.overlay_dir` escapes project root via path traversal | Tampering | LOW | Accept: Phase 1's submit already validates `..` traversal (REG-04); `JobSpec` receives a validated `Path` object; backend trusts the caller's construction |
| T-02-01-S04 | `JobState.BUDGET_KILLED` used by Phase 2 backends prematurely | Elevation of Privilege | LOW | Accept: D-53 explicitly reserves this value for Phase 4; Phase 2 tests assert no backend produces it; if a Phase 2 backend returns it, test_contract will fail |
