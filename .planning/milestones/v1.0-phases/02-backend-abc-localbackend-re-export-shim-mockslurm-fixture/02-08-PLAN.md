# Plan 02-08: `automil cancel` + `automil resubmit` CLI Commands + Integration Tests

## Objective

Implement `automil cancel <node_id>` and `automil resubmit <node_id>` CLI commands that route through the Backend ABC. Cancel terminates a running experiment and archives its state; resubmit spawns a new node from a terminal node's archived overlay. Both commands are tested end-to-end against `MockSLURMBackend` with a synthetic graph fixture, covering happy paths and all error cases specified in D-66/D-67.

Per PATTERNS.md §7, both commands use `_get_node_or_die` for node lookup and the "Refusing to X: what. why. suggestion." ClickException format. Per PATTERNS.md §8, `BACKENDS` is lazily imported inside the Click function body to prevent circular imports.

Per D-70 point 5, the integration test uses a pre-populated synthetic `graph.json` with one running node (with `metadata.backend = "mock_slurm"`, `metadata.opaque_id`, `metadata.submitted_at` set), one completed node, and one crashed node. The test starts a `MockSLURMBackend` with the same opaque state to simulate a live backend.

Purpose: Without cancel/resubmit, the agent loop cannot self-correct misconfigured or crashed batches.

Output: `cancel.py`, `resubmit.py`, `__init__.py` update, `test_cli_cancel_resubmit.py`. All tests green.

- T-02-08-02: Implement `src/automil/cli/cancel.py`
  - Module docstring: references CLI-03, D-66
  - `from __future__ import annotations`
  - `import time`, `from datetime import datetime`, `import logging`, `from pathlib import Path`
  - `import click`
  - `from automil.cli import main`
  - `from automil.cli._helpers import _find_automil_dir, _find_git_root` (match exact names from read pass)
  - `from automil.cli.lifecycle._shared import _get_node_or_die`
  - `logger = logging.getLogger(__name__)`
  - `@main.command("cancel")` + `@click.argument("node_id")` + `@click.option("--timeout", default=30, ...)` decorator stack
  - Implement D-66 workflow steps 1–8:
    1. `adir = _find_automil_dir()`; lazy import BACKENDS inside function body
    2. `node = _get_node_or_die(adir, node_id)` — hard-fail if unknown (format per PATTERNS.md §7)
    3. If `node.get("status") != "running"`: raise ClickException with "Refusing to cancel: node {id!r} is in state {state!r}, not 'running'. Only running experiments can be cancelled. Use `automil status`."
    4. `backend_name = node.get("metadata", {}).get("backend", "local")` — D-76 default fallback (legacy nodes default to "local")
    5. Read `running/<node_id>.json` to get the live `opaque_id` and `submitted_at` — these are written by the daemon at launch time, NOT by submit (D-76: PID unknown until launch). If `running/<node_id>.json` does not exist, raise ClickException("Refusing to cancel: no running spec at {path}. Node may have already finished — try `automil status`."). Schema check: `opaque_id` must be a non-empty string; if missing, raise ClickException("Running spec at {path} is missing 'opaque_id' — corrupted state, manage the process manually.")
    6. Resolve `BackendClass = BACKENDS.get(backend_name)`; if None: raise ClickException("Unknown backend {backend_name!r}; available: {sorted(BACKENDS.keys())}")
    7. Instantiate `backend = BackendClass(project_root=_find_git_root(), automil_dir=adir)`
    8. Reconstruct `handle = JobHandle(node_id=node_id, backend=backend_name, opaque_id=running_spec["opaque_id"], submitted_at=running_spec["submitted_at"])` — opaque_id from running/<id>.json (W-03 fix per plan-check iter 1)
    9. `backend.cancel(handle)` — fire-and-forget
    10. Poll loop: wait up to `timeout` seconds for `CANCELLED` state (D-66 step 6); `time.sleep(1.0)` tick
    11. If timeout: raise ClickException with current state + diagnostic
    12. Atomic graph update (D-66 step 7): `node["status"] = "cancelled"`, `node.setdefault("metadata", {})["cancelled_at"] = datetime.now().isoformat()`, `node["metadata"]["cancel_reason"] = "cli"`, `graph.save()` using `ExperimentGraph` API
    13. Move `running/<node_id>.json` → `archive/<node_id>/` (D-66 step 8)
    14. `click.echo(f"Cancelled {node_id}.")`

- T-02-08-03: Implement `src/automil/cli/resubmit.py`
  - Module docstring: references CLI-04, D-67
  - Same import structure as cancel.py
  - `@main.command("resubmit")` + `@click.argument("node_id")`
  - Implement D-67 workflow steps 1–7:
    1. `adir = _find_automil_dir()`; lazy import BACKENDS
    2. `node = _get_node_or_die(adir, node_id)` — hard-fail if unknown
    3. Terminal states = `{"completed", "crashed", "cancelled"}`; if `node.get("status") not in terminal_states`: raise ClickException("Refusing to resubmit: node {id!r} is in state {state!r}. Only terminal nodes (completed|crashed|cancelled) can be resubmitted.")
    4. Read `archive/<node_id>/` directory — find overlay files (all files except `result.json` and `*_running_spec.json`)
    5. Generate new node_id via `graph.new_node_id()` or equivalent (check ExperimentGraph API from read pass)
    6. Build `new_spec = JobSpec(node_id=new_node_id, ...)` mapping from the archived spec; `backend_name = node.get("metadata", {}).get("backend", "local")`
    7. Resolve and instantiate backend; `handle = backend.submit(new_spec)`
    8. Insert new graph node: `parent_id = node.get("parent_id")`, `metadata.resubmitted_from = node_id`
    9. `graph.save()`
    10. `click.echo(new_node_id)` — print new node_id to stdout for operator capture (D-67 step 7)
