<objective>
Implement `automil revert-baseline` (CLI-02), the second of the six lifecycle commands. Per D-42, the command runs `git checkout <base_commit> -- <protected_paths>` for paths matching `registry.protected`. The "anti-protected = implicit editable" semantic means the operator decides what's editable by NOT listing it in `protected`; revert-baseline resets only the protected paths, leaving editable code (the consumer's variant modules + any other agent-touched paths) untouched.

**Critical safety property** (D-42 + Leo's "never blind-checkout after submit" memory + the project's `feedback_never_blind_checkout.md` learning): the command MUST stash uncommitted changes before the checkout. Without the stash, an operator who runs `revert-baseline` while in the middle of an experiment overlay loses uncommitted work silently. The stash name format is `automil-revert-<timestamp>` and the stash name is printed to stdout so the operator can `git stash list` and `git stash pop` if needed.

Per CLI-02 idempotence (REG-04 + general workflow expectation), running on a clean working tree is a no-op: no stash is created, no checkout is run, the command exits 0 with a "nothing to do" message.

The command derives `base_commit` from the most recent executed-status node's `base_commit` field in `graph.json`. If multiple executed nodes exist with different base_commits (rare; happens when the operator ran `automil submit` before AND after a non-trivial git rebase), the most-recent-by-creation-timestamp wins. Hard-fail with operator-friendly message if no graph.json, no executed nodes, or base_commit is invalid (`git rev-parse <sha>` fails).

Per the production-grade quality bar:
- `--help` text explains when to use the command (after the agent has accumulated edits to shared library files).
- Stash name printed to stdout BEFORE the checkout, so even if the checkout fails, the operator sees the stash name and can recover.
- Subprocess error messages are surfaced with their stderr content (don't swallow git's own error).

```python
"""revert-baseline command: reset registry.protected paths to base_commit (CLI-02 / D-42).

Safety: MANDATORY pre-stash before any checkout. Leo's "never blind-checkout
after submit" memory: `git checkout -- <file>` silently destroys uncommitted
work. This command stashes first, prints the stash name, then checks out.
"""
from __future__ import annotations

import json
import logging
import subprocess
from datetime import datetime
from pathlib import Path
from typing import Optional

import click

from automil.cli import main
from automil.cli._helpers import _find_automil_dir, _find_git_root
from automil.cli.lifecycle._shared import _load_registry_or_die

logger = logging.getLogger(__name__)


def _latest_executed_base_commit(adir: Path) -> Optional[str]:
    """Return the most-recent executed node's base_commit, or None if no graph.

    "Most recent" = lexicographic max on `created_at` ISO-8601 string (works
    because ISO-8601 sorts correctly).
    """
    graph_path = adir / "graph.json"
    if not graph_path.exists():
        return None
    try:
        graph = json.loads(graph_path.read_text())
    except json.JSONDecodeError:
        return None
    executed_with_base: list[tuple[str, str]] = []  # (created_at, base_commit)
    for node in graph.get("nodes", {}).values():
        if node.get("type") == "executed":
            base = node.get("base_commit")
            ts = node.get("created_at", "")
            if base:
                executed_with_base.append((ts, base))
    if not executed_with_base:
        return None
    executed_with_base.sort()
    return executed_with_base[-1][1]


def _has_uncommitted_changes(git_root: Path) -> bool:
    """`git status --porcelain` returns non-empty iff dirty (staged or unstaged)."""
    result = subprocess.run(
        ["git", "status", "--porcelain"],
        cwd=git_root, capture_output=True, text=True, check=True,
    )
    return bool(result.stdout.strip())


@main.command("revert-baseline")
def revert_baseline():
    """Reset registry.protected paths to their base_commit state.

    Workflow: when the agent has accumulated edits to shared library files
    that should NOT have been touched (registry.protected violation),
    run `automil revert-baseline` to git-checkout those paths back to clean.
    STASHES uncommitted changes first (Leo's "never blind-checkout" memory) —
    the stash name is printed to stdout so you can recover via `git stash pop`.

    Idempotent: running on a clean tree is a no-op.

    Hard-fails if:
      - no graph.json (cannot determine base_commit)
      - no executed nodes (cannot determine base_commit)
      - registry.protected is empty (nothing to revert)
      - git stash or git checkout fails (stderr surfaced)
    """
    adir = _find_automil_dir()
    git_root = _find_git_root()
    cfg = _load_registry_or_die(adir)

    # Validate inputs.
    if not cfg.protected:
        raise click.ClickException(
            "registry.protected is empty in automil/config.yaml. "
            "Nothing to revert. Edit the config to list paths the agent "
            "must not touch (e.g., 'benchmarks/lib/CLAM/**')."
        )

    base_commit = _latest_executed_base_commit(adir)
    if base_commit is None:
        raise click.ClickException(
            "Cannot determine base_commit: no executed nodes in "
            "automil/graph.json with a recorded base_commit field. "
            "Run at least one experiment via `automil submit` first."
        )

    # Verify base_commit is a valid SHA.
    rp = subprocess.run(
        ["git", "rev-parse", "--verify", base_commit],
        cwd=git_root, capture_output=True, text=True,
    )
    if rp.returncode != 0:
        raise click.ClickException(
            f"base_commit {base_commit!r} is not a valid git SHA (resolved "
            f"from the most recent executed node). Check your git history; "
            f"the commit may have been rewritten or pruned."
        )

    # Idempotence check: clean tree at protected paths -> no-op.
    status = subprocess.run(
        ["git", "status", "--porcelain", "--", *list(cfg.protected)],
        cwd=git_root, capture_output=True, text=True, check=True,
    )
    if not status.stdout.strip():
        click.echo("revert-baseline: protected paths already clean; nothing to do.")
        return

    # MANDATORY pre-stash (D-42 + Leo's never-blind-checkout memory).
    if _has_uncommitted_changes(git_root):
        ts = datetime.now().strftime("%Y%m%d-%H%M%S")
        stash_name = f"automil-revert-{ts}"
        stash_result = subprocess.run(
            ["git", "stash", "push", "--include-untracked", "-m", stash_name],
            cwd=git_root, capture_output=True, text=True,
        )
        if stash_result.returncode != 0:
            raise click.ClickException(
                f"`git stash push` failed: {stash_result.stderr.strip()}. "
                f"Resolve the conflict and retry, or revert manually with "
                f"`git checkout -p`."
            )
        click.echo(f"Stashed uncommitted changes as {stash_name!r}.")
        click.echo(f"  Recover via: git stash list  &&  git stash pop")
    else:
        click.echo("Working tree clean; no stash needed.")

    # Now safe to checkout protected paths.
    co = subprocess.run(
        ["git", "checkout", base_commit, "--", *list(cfg.protected)],
        cwd=git_root, capture_output=True, text=True,
    )
    if co.returncode != 0:
        raise click.ClickException(
            f"`git checkout {base_commit[:8]}` for protected paths failed: "
            f"{co.stderr.strip()}. Inspect with `git status` and resolve manually."
        )

    click.echo(f"Reverted protected paths to base_commit {base_commit[:8]}.")
    click.echo(f"  Patterns: {list(cfg.protected)}")
```

**Note on git checkout glob behaviour:** `git checkout <sha> -- 'benchmarks/lib/CLAM/**'`
won't expand the glob (git's pathspec is not shell-glob). For glob patterns, use
`git checkout <sha> -- :(glob)benchmarks/lib/CLAM/**` syntax, OR enumerate the
matching files via `git ls-files --modified --others -- <patterns>`. The
implementation passes the patterns directly; if a consumer's protected list uses
shell-globs that git pathspec doesn't expand, the operator gets a clear error
("did not match any file(s) known to git"). For Phase 1 this is acceptable —
the synthetic mini-consumer uses literal paths; the CCRCC follow-up can switch
to `:(glob)` syntax if needed (documented in `<output>` notes for downstream).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator -> revert-baseline -> git checkout | The destructive operation (`git checkout` on protected paths) ONLY runs after a successful pre-stash. The stash is an in-band safety net the operator can use to recover. |
| graph.json -> base_commit derivation | `_latest_executed_base_commit` reads graph.json; corrupt JSON falls through to None and triggers the hard-fail "no executed nodes" path (graceful). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-40 | T (tampering) | Blind-checkout destroys uncommitted work | mitigate (HIGH) | MANDATORY pre-stash via `git stash push --include-untracked -m automil-revert-<ts>` BEFORE the checkout. The stash name is printed; recovery instructions printed. Test `test_mandatory_stash_created` + `test_uncommitted_non_protected_also_stashed` + `test_untracked_file_included_in_stash` all enforce. Without this mitigation, the agent's accumulated work is destroyed silently — Leo's memory is explicit on this. |
| T-01-41 | T (tampering) | Stash silently fails (e.g., merge conflict in stash apply later) | accept | The stash name is printed BEFORE the checkout, so the operator has a recovery handle even if `git stash apply` later runs into conflicts. The plan does NOT auto-pop the stash — that's the operator's choice. |
| T-01-42 | T (tampering) | git stderr swallowed | mitigate | Both `git stash push` and `git checkout` capture stderr and surface it in the click.ClickException message. Test verifies stderr text appears in the failure output (an explicit assertion). |
| T-01-43 | I (information disclosure) | base_commit hash printed | accept | Same as T-01-26: SHAs are not secrets. |

**HIGH severity item:** T-01-40 (blind-checkout). Three dedicated tests + the --help text mentioning the stash safety property. Plan 01-10's commit cannot land without all three tests green.

**State-transition recovery:** If the command crashes BETWEEN the stash and the checkout (e.g., SIGKILL), the stash exists in the operator's stash list with the `automil-revert-` prefix; the operator can `git stash pop`. If it crashes during the checkout, git rolls back the partial state automatically (atomic per-file). No silent corruption.
