  <how-to-verify>
    Step-by-step pilot procedure:

    1. **Pick fresh cells.** Run `automil cell list` to see all current cells with their `consumed_seconds` and `status`. Pick 3 CCRCC + 2 CLWD where status='active' AND consumed_seconds is low (<<budget). Note their cell_ids.

    2. **Confirm node_0176 is portable.** If REG-08 (Phase 1) has shipped, `automil apply node_0176` should work. If not, manually identify node_0176's overlay files (likely the dirty edits to clam_mb model + ce_smooth loss + sam_lookahead policy).

    3. **Register the manifest.** For each fresh cell, you'll create a NEW candidate of node_0176's parent. The manifest declares all 5 fresh cells as held-out:
       ```bash
       automil gate register-manifest <node_0176's parent_id> \
           --K 3 \
           --p-threshold 0.05 \
           --bootstrap-reps 1000 \
           --strategy operator-curated \
           --held-out-cells "<5 cell tuples>"
       ```

    4. **Submit candidates.** For each held-out cell, submit a candidate that applies node_0176-equivalent changes. The orchestrator's per-cell cap will run them; results appear as the cells finish.

    5. **Run --calibrate.** When all 5 are complete, pick ONE candidate that has all 5 cells' results available (or use the orchestrator's auto-spawned gate-eval children of any nominated candidate). Run:
       ```bash
       automil nominate <candidate_id>
       automil promote <candidate_id> --calibrate
       cat automil/archive/<candidate_id>/gate_evaluation.jsonl
       ```
       The first JSONL line has the per-cell results array.

    6. **Fill in `.planning/phase-05-calibration.md`.** Replace `_tbd_` with real values. Compute the recommended K based on:
       - How many cells delta > 0 (wins)?
       - Does Wilcoxon p ≤ 0.05/K_chosen pass?
       - Is BCa CI lower bound > 0?

    7. **Lock K (if different from framework default).** Edit `src/automil/templates/config.yaml.j2`:
       ```yaml
       gate:
         K:  <chosen K>    # locked by Phase 5 calibration; see .planning/phase-05-calibration.md
       ```
       Commit with message `gate: lock K=<N> from Phase 5 calibration pilot`.

    8. **Sign off.** Check the boxes in the calibration doc; commit; update STATE.md to mark Phase 5 "calibrated."
