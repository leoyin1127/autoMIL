<objective>
Defend Pitfall 6c (held-out leak into agent loop). Two surfaces leak today:
1. **Trajectory** — agent stdout/stderr capture might surface `node_NNNN` IDs that happen to be gate-eval children. Without redaction, these IDs land in trajectory.jsonl and could be retrieved into prompt context, defeating pre-registration.
2. **Rank output** — `automil rank` shows top proposals to the agent; gate-eval children must NOT appear there.

Fix both at the read site (rank filters on output; redactor replaces in-event strings) AND tag at the write site (Wave 1 plan 03 added `JobSpec.metadata` for the writer; Wave 3 plan 06 will set `held_out=true` on submit). The lookup is mtime-cached so a fresh gate-eval submit invalidates within one fs tick without per-event graph reads.

Purpose: Without this isolation, the D-149 Pitfall-6 anti-acceptance test (plan 11) cannot pass — the test asserts trajectory.jsonl contains zero held-out IDs. This plan delivers the redaction primitive that test depends on.
Output: redactor extension + rank filter + 2 new test files (no changes to existing redactor or rank tests).

    ```python
    @main.command()
    @click.option("--n", default=6, help="Number of proposals to return")
    @click.option("--max-per-branch", default=2, help="Max proposals per branch")
    @click.option(
        "--include-held-out",
        is_flag=True,
        default=False,
        help="Include held-out gate-eval nodes (D-139; logs WARNING; do NOT use during agent search loop).",
    )
    def rank(n: int, max_per_branch: int, include_held_out: bool):
        """Show top-ranked proposals from the experiment graph."""
        adir = _find_automil_dir()
        graph_path = adir / "graph.json"

        if not graph_path.exists():
            click.echo("No graph.json found. Run some experiments first.")
            return

        from automil.graph import ExperimentGraph
        graph = ExperimentGraph(path=str(graph_path))

        # D-139: held-out isolation — filter unless operator explicitly opts in.
        if include_held_out:
            import logging as _logging
            _logging.getLogger(__name__).warning(
                "rank --include-held-out: held-out cell composites now visible; "
                "this MUST NOT be used during the agent search loop (D-139)."
            )
        else:
            graph._data["nodes"] = {
                k: v
                for k, v in graph.nodes.items()
                if not (
                    isinstance(v, dict)
                    and v.get("metadata", {}).get("held_out", False)
                )
            }

        graph.recalculate_scores()
        proposals = graph.rank_proposals(n=n, max_per_branch=max_per_branch)

        if not proposals:
            click.echo("No proposals available. Time to brainstorm!")
            return

        click.echo(f"Top {len(proposals)} proposals:\n")
        for i, node in enumerate(proposals, 1):
            node_id = node["id"]
            parent = node.get("parent_id", "root")
            desc = node.get("description", "")
            score = node.get("potential", 0)
            click.echo(f"  {i}. [{node_id}] (parent: {parent}, score: {score:.4f})")
            click.echo(f"     {desc}")
            click.echo()
    ```

    Note: the filter mutates `graph._data["nodes"]` rather than re-parenting via `graph.nodes` — that matches the existing pattern of how rank reads the dict. No persistence side-effect (graph.save() is NEVER called by rank).
