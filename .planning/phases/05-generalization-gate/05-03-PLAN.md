<objective>
Extend the Backend ABC's `JobSpec` with a `metadata: tuple[tuple[str, str], ...] = ()` field so gate-eval submissions (Wave 3 evaluate.py) can pass `gate_eval`, `held_out`, `gate_parent_node`, `cell_id`, and `edge_type` keys through Backend.submit() — the SAME path the agent uses for normal submits (GTE-03 explicit: "Backend.submit, NOT a parallel mechanism").

This is a NON-BREAKING change: tuple field with default `()`, kw-only via positioning at end of dataclass. All existing JobSpec construction sites continue to work.

Purpose: Without this field, gate/evaluate.py would have to bypass Backend.submit() and write to queue/<id>.json directly — a Pitfall 6 violation (parallel mechanism). One-field ABC extension preserves the contract test surface.
Output: src/automil/backends/base.py + LocalBackend.submit() + MockSLURMBackend.submit() + contract test.
</objective>

| T-05-03-02 | Information Disclosure | metadata logged to queue/<id>.json which is gitignored but readable on disk | accept | queue specs are framework-internal; no PII; held-out IDs are not secrets per V8 ASVS partial classification |

**Note for executor:** Task 1 currently merges spec.metadata AFTER `queue_spec.setdefault("metadata", {})["backend"] = "local"`. Per T-05-03-01 above, the backend stamp must NOT be overridable by caller-provided metadata. Re-order so the backend stamp is set LAST:
```python
# Phase 5 D-140: merge spec.metadata FIRST
for k, v in spec.metadata:
    queue_spec.setdefault("metadata", {})[k] = v
# Then backend stamp wins — caller cannot override framework key
queue_spec.setdefault("metadata", {})["backend"] = "local"
```
