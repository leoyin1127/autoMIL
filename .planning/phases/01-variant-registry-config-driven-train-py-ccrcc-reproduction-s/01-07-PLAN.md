Per D-46, `automil check` Phase 1 additions are: protected-files clean, registry consistency, repro-manifest warn — env-required and runtime-asset deferred to Phase 8 / Phase 3 respectively.

Without the submit-hook, an agent could overlay an edit to `benchmarks/lib/CLAM/models/model_clam.py` and the framework would never notice. Without the check extensions, `automil check` would say "all good" while the registry was inconsistent.

```python
from automil.registry.config import load_registry_config

reg_cfg = load_registry_config(adir)

# Helper: identify variant-module file paths so we run the validator chain on them.
def _is_variant_module_path(rel_path: str) -> bool:
    # rel_path is relative to git_root.
    parts = Path(rel_path).parts
    if "variants" not in parts:
        return False
    # Path looks like .../variants/<parent_or_kind>/<name>.py
    idx = parts.index("variants")
    return (
        idx + 2 < len(parts)
        and parts[idx + 2].endswith(".py")
        and not parts[idx + 2].startswith("_")
        and parts[idx + 2] != "__init__.py"
    )
```

```python
                + "\n      Run `automil revert-baseline` to reset, or commit the changes "
                "to a variant module via `automil port-variant <node_id>`."
            )
    except (subprocess.TimeoutExpired, subprocess.CalledProcessError):
        warnings.append("Could not run `git status` for protected files — git may not be on PATH.")

# Registry consistency (D-46): every <variants_root>/<*>/*.py imports cleanly +

    for path, exc_str in scan_result.failed:
        issues.append(f"Variant module {path} failed import: {exc_str}")

    for var_path in scan_result.imported:
        manifest_path = var_path.with_suffix(".json")

        ok, reason = manifest.cross_check_with_module(var_path)
        if not ok:
            issues.append(
                f"Manifest {manifest_path.name} mismatches docstring of {var_path.name}: {reason}"
```
