<objective>
Ship the framework's variant type system: three sibling Abstract Base Classes (`ModelVariant`, `LossVariant`, `PolicyVariant`) and the frozen `VariantSpec` provenance record. This is the foundation every subsequent Phase 1 plan builds on — Plan 01-02's registry singleton stores `type[ModelVariant]` etc., Plan 01-04's interface validator does `isinstance(cls, type) and issubclass(cls, ModelVariant)` checks, Plan 01-08's lifecycle commands construct `VariantSpec` instances when porting variants.

Per D-21, three sibling ABCs (NOT one polymorphic ABC) — a `LossVariant` cannot satisfy `forward(features, coords) -> AggregatorOutput`, so a single ABC violates Liskov + interface segregation. Per D-22, `VariantSpec` is `@dataclass(frozen=True)` because it is a registry key + provenance record; accidental mutation must raise `FrozenInstanceError`. Per D-23, `kind` is `Literal["model", "loss", "policy"]` exhaustively for Phase 1 — `recipe` (composition) and `inference` (gate-time) are deferred. Per D-24, `AggregatorOutput` is NOT introduced — `ModelVariant.forward` returns whatever the parent model returns, and Phase 8's `result.json` JSON-Schema is the right place to formalise framework-level output contracts.

Purpose: Establish the typed surface every variant module subclasses against. Without this, Plan 01-02's registry has nothing to key on, Plan 01-04's validator has no ABC to walk, and Plan 01-08's `port-variant` cannot produce a manifest. This is the keystone keystone.

    """ModelVariant ABC — per-parent model variants (REG-01 / D-21).

    Subclasses live under `<consumer>/<dataset>/automil/variants/<parent>/<name>.py`
    and are registered via the `@register` decorator (Plan 01-02). They are
    instantiated by `train.py` after the registry resolves the variant's class.
    """
    from __future__ import annotations

    import logging
    from abc import ABC, abstractmethod
    from typing import Any, Optional, TYPE_CHECKING

    if TYPE_CHECKING:
        # Type-only import — keeps this module loadable without torch installed
        # (D-24: framework does NOT depend on torch; the synthetic mini-consumer
        # in Plan 01-12 is torch-free, and Phase 8 / DEC-03 owns the framework
        # output type).

    class ModelVariant(ABC):
        """Per-parent model variant.

        Subclasses MUST implement `forward`. Return shape is whatever the parent
        model returns (D-24) — Phase 1 does NOT introduce an `AggregatorOutput`
        framework type. CLAM, for example, returns `(logits, Y_prob, Y_hat,
        instance_dict)`; sklearn-iris (Phase 8 / DEC-02) returns its own shape.
        The parent's wrapper code in `train.py` is responsible for any conversion.
        """

        @abstractmethod
        def forward(self, features: Any, coords: Optional[Any] = None) -> Any:
            """Forward pass on a bag of feature vectors.

            Args:
                features: Tensor of shape (N, D) where N=instances, D=feature dim.
                coords: Optional spatial coordinates of shape (N, 2) for
                        positional models. May be None for parents that ignore
                        spatial structure.

            Returns:
                Whatever the parent model returns (D-24).
            """

        def instance_attention(self, features: Any, coords: Optional[Any] = None) -> Optional[Any]:
            """Optional per-instance attention output. Default returns None.

            Variants that compute per-instance attention (CLAM, AB-MIL, DSMIL)
            override this to surface the attention weights for downstream
            interpretability / clustering / gate-eval tooling.
            """

    """PolicyVariant ABC — parent-agnostic training-policy variants (REG-01 / D-21).

    Policies wrap the optimizer / scheduler to implement strategies like SAM,
    Lookahead, gradient accumulation, etc. The default `step` delegates to
    `opt.step()` so non-SAM-style policies need only override `wrap_optimizer`.
    """
    from __future__ import annotations

    import logging
    from abc import ABC, abstractmethod
    from typing import Any

    logger = logging.getLogger(__name__)


    class PolicyVariant(ABC):
        """Parent-agnostic training-policy variant."""

        @abstractmethod
        def wrap_optimizer(self, opt: Any) -> Any:
            """Wrap (or replace) the optimizer; return the wrapped instance."""

        def wrap_scheduler(self, sched: Any) -> Any:
            """Optional scheduler wrapping. Default returns the input unchanged."""
            return sched

        def step(self, loss: Any, opt: Any) -> None:
            """Default step: delegate to opt.step() after backward.

            SAM-style two-step policies override this to implement `first_step`
            + `second_step`. The default works for vanilla / Lookahead-only /
            gradient-accumulation policies.
            """
            opt.step()
