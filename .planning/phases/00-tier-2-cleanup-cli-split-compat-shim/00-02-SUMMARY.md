### New `_load_dotenv` body

```python
def _load_dotenv(self) -> None:
    """Load .env files from the project root into os.environ.

    Worktrees are detached git checkouts and don't contain .env
    (which is typically gitignored). Loading here ensures the
    orchestrator's child processes inherit the variables.

    Uses python-dotenv so quoted values, the ``export`` prefix, and
    inline ``# comments`` after unquoted values are handled
    correctly (CLN-03; see CONCERNS.md §"Naive .env parser").
    Pre-existing entries in ``os.environ`` are preserved
    (``setdefault`` semantic) — the shell wins over the file.
    """
    candidates = [
        self.project_root / ".env",
        self.project_root / "benchmarks" / ".env",
    ]
    for env_file in candidates:
        if not env_file.is_file():
            continue
        parsed = dotenv_values(env_file)
        for key, value in parsed.items():
            if value is None:
                continue
            # Don't override existing env vars (preserves prior semantic).
            if key not in os.environ:
                os.environ[key] = value
                logger.debug("Loaded env var %s from %s", key, env_file)
```

## Decisions Made

- **Single commit boundary per D-20** rather than RED test commit + GREEN impl commit. The plan declared `tdd="true"` but D-20 explicitly mandates one commit per CLN item. TDD was honored by writing tests first and confirming RED before implementing, but no separate `test:` commit was issued.
- **Skip `None` values from `dotenv_values`** — the python-dotenv API returns `None` for keys it parses as having no value (e.g., `KEY=` with no RHS, or unset interpolations). Without this guard `os.environ[key] = None` would raise. The legacy parser skipped empty-key lines via `if not key`, so this is the equivalent guardrail one layer up.
