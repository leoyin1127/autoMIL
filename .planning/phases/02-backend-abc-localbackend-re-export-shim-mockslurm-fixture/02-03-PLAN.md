# Plan 02-03: Extend `cli/submit.py` to Persist `metadata.backend`

## Objective

Add a 3-line extension to `src/automil/cli/submit.py` that writes `metadata.backend = "<name>"` to the `queue/<id>.json` spec at submit time. This is the prerequisite for Plan 02-08's `cancel.py` and `resubmit.py` — both commands must reconstruct the `JobHandle` by reading `node.metadata.backend` to know which `BACKENDS[name]` to dispatch to.

Per D-76, the backend name comes from `automil/config.yaml: backend.name` (default `"local"`). `opaque_id` is NOT written at submit time (the daemon does not yet know the PID). Backward compat: `cancel.py` will treat missing `metadata.backend` as `"local"` so legacy nodes submitted before Phase 2 still cancel correctly.

This plan has NO dependency on Plans 02-01/02-02 — it only writes a string field to the existing queue spec dict. It is Wave 1 because it is a pure extension to existing code with no dependency on backend objects.

Purpose: `cancel.py` in Plan 02-08 will fail with `KeyError: 'backend'` if this field is not present.

Output: 3-line addition to `submit.py`. All 387 tests green.

## Implementation Tasks

- T-02-03-01: Read `src/automil/cli/submit.py` in full before touching anything
  - Identify where the `metadata` dict is constructed/written into the queue spec
  - Identify where `automil/config.yaml` is read (to find the existing `config_loader` pattern)
  - Note the exact variable name for the queue spec dict (likely `spec` or `queue_spec`)

- T-02-03-02: Add `metadata.backend` to the queue spec dict
  - In the section where `metadata` fields are populated (near `config_hash`, `parent_id`, `techniques`), add:
    ```python
    # D-76: write backend name so cancel/resubmit can dispatch to correct backend
    _backend_name = _config.get("backend", {}).get("name", "local")
    spec.setdefault("metadata", {})["backend"] = _backend_name
    ```
  - `_config` is whatever variable name holds the loaded `automil/config.yaml` dict — match the existing variable name in submit.py exactly; do NOT introduce a new config load
  - If `automil/config.yaml` does not have a `backend` key, the `.get("backend", {}).get("name", "local")` chain returns `"local"` — satisfying D-76's backward compat rule
  - Do NOT add `opaque_id` or `submitted_at` here — D-76 explicitly says these are daemon-side (written when `_launch_experiment` returns)
  - Do NOT write `backend` to the `graph.json` node at submit time — only to the `queue/<id>.json` spec. The graph node gets `metadata.backend` populated later when the daemon picks up the job OR when `cancel.py` reads it from the running spec. The integration test in Plan 02-08 uses a synthetic graph fixture with the field pre-populated.

- T-02-03-03: Verify no regressions
  - `uv run pytest tests/ -x -q` — must exit 0 with 387 tests
  - `grep -n "metadata.*backend\|backend.*name" src/automil/cli/submit.py` — must show the new line
  - Single conventional commit: `feat(cli/submit): write metadata.backend to queue spec (D-76, BCK-01 prereq)`
