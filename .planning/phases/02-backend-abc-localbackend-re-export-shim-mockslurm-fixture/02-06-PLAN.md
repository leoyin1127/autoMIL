# Plan 02-06: `MockSLURMBackend` — Eventual-Consistency Test Fixture

## Objective

Implement `MockSLURMBackend` at `src/automil/backends/mock_slurm.py`. This is the second concrete backend implementation — existing alongside Plan 02-05's `LocalBackend` — that drives the "test ABC against ≥2 implementations before locking" anti-acceptance criterion (D-70 / Phase 2 success criterion #3).

Per D-62, `MockSLURMBackend` simulates SLURM's eventual-consistency semantics using `threading.Timer` + `threading.Event`. It does NOT execute real commands (D-63) — the job outcome is determined by `JobSpec.command` content. Per RESEARCH.md §6, the `threading.Lock` guards `_jobs` dict; the `threading.Event` for `cancel_requested` is atomic; daemon timers are `daemon=True` to prevent test hangs.

Per D-69, `MockSLURMBackend` is NOT auto-imported in `backends/__init__.py`. It is registered only when the test (or user) explicitly imports `mock_slurm`. This prevents test fixtures leaking into production config.

Purpose: Without MockSLURM, the contract test (Plan 02-07) can only run against one backend and cannot prove the ABC is general. Phase 2's core anti-acceptance criterion fails.

Output: `mock_slurm.py` (~200 lines). No change to existing tests. No new tests in this plan (contract test is Plan 02-07).

- T-02-06-01: Implement `src/automil/backends/mock_slurm.py`
  - Module docstring: references BCK-03, D-62, D-63, D-69; note "NOT auto-registered in backends/__init__.py — import explicitly in tests (D-69)"
  - `from __future__ import annotations`
  - Imports: `import json`, `import logging`, `import threading`, `import time`, `from dataclasses import dataclass, field`, `from pathlib import Path`, `from typing import Iterator, Optional`
  - `from automil.backends.base import Backend, JobHandle, JobSpec, JobState`
  - `from automil.backends import register`
  - `logger = logging.getLogger(__name__)`
  - Define `_MockJob` dataclass (NOT frozen — needs mutable `state` and `log_buffer`):
    ```python
    @dataclass
    class _MockJob:
        handle: JobHandle
        state: JobState
        cancel_requested: threading.Event = field(default_factory=threading.Event)
        log_buffer: list[str] = field(default_factory=list)
        timer: Optional[threading.Timer] = field(default=None, repr=False)

        def _to_json(self) -> dict:
            """Serialise to JSON (excludes cancel_requested and timer — runtime-only)."""
            return {
                "node_id": self.handle.node_id,
                "backend": self.handle.backend,
                "opaque_id": self.handle.opaque_id,
                "submitted_at": self.handle.submitted_at,
                "state": self.state.value,
                "log_buffer": self.log_buffer,
            }

        @classmethod
        def _from_json(cls, data: dict) -> "_MockJob":
            """Deserialise from JSON. PENDING/RUNNING jobs at restart → CRASHED (RESEARCH.md §7)."""
            handle = JobHandle(
                node_id=data["node_id"],
                backend=data["backend"],
                opaque_id=data["opaque_id"],
                submitted_at=data["submitted_at"],
            )
            state = JobState(data["state"])
            # Jobs that were PENDING/RUNNING at crash-time cannot resume timers
            if state in (JobState.PENDING, JobState.RUNNING):
                state = JobState.CRASHED
                logger.warning(
                    "MockSLURMBackend restart: job %s was in state %s at shutdown; "
                    "marking as CRASHED (no timer resumption possible)",
                    data["node_id"], data["state"],
                )
            return cls(handle=handle, state=state, log_buffer=data.get("log_buffer", []))
    ```
  - Define `@register("mock_slurm")` + `class MockSLURMBackend(Backend)`:
    - `__init__(self, poll_lag_seconds: float = 5.0, state_file: Optional[Path] = None)`:
      - Store `self._poll_lag = poll_lag_seconds`
      - `self._jobs: dict[str, _MockJob] = {}` (opaque_id → job)
      - `self._counter = 0`
      - `self._lock = threading.Lock()`
      - `self._state_file = state_file`
      - If `state_file` and `Path(state_file).exists()`: `self._load_state()`
    - `submit(self, spec: JobSpec) -> JobHandle`:
      - Under `_lock`: `self._counter += 1; opaque_id = f"{self._counter}.0"`
      - Construct `handle = JobHandle(node_id=spec.node_id, backend="mock_slurm", opaque_id=opaque_id, submitted_at=time.time())`
      - Create `job = _MockJob(handle=handle, state=JobState.PENDING)`
      - Under `_lock`: `self._jobs[opaque_id] = job`
      - Launch timer chain (see RESEARCH.md §6 code skeleton verbatim for the nested `_transition` + `_finish` pattern)
      - The `_finish` function uses `spec.command` content to determine terminal state (D-63): `if "--crash" in " ".join(spec.command): CRASHED else: COMPLETED`
      - `job.log_buffer.append("mock: job started")` in `_transition`; `job.log_buffer.append(f"mock: job terminal ({job.state})")` in `_finish`
      - `self._persist_state()`
      - Return `handle`
    - `poll(self, handle: JobHandle) -> JobState`:
      - Under `_lock`: get `job = self._jobs.get(handle.opaque_id)`
      - If `job is None`: raise `ValueError(f"Unknown job: {handle.opaque_id}")`
      - Return `job.state` (pure snapshot, no side effects)
    - `cancel(self, handle: JobHandle, signal: Optional[int] = None) -> None`:
      - Under `_lock`: get `job = self._jobs.get(handle.opaque_id)`
      - If `job is None`: return (fire-and-forget: unknown jobs silently ignored per D-57)
      - `job.cancel_requested.set()` — atomic (threading.Event)
      - Return None immediately
    - `list_running(self) -> list[JobHandle]`:
      - Under `_lock`: return `[j.handle for j in self._jobs.values() if j.state in (JobState.PENDING, JobState.RUNNING)]`
    - `log_iter(self, handle: JobHandle) -> Iterator[str]`:
      - `TERMINAL = {JobState.COMPLETED, JobState.CRASHED, JobState.CANCELLED, JobState.BUDGET_KILLED}`
      - `job = self._jobs.get(handle.opaque_id)` (no lock needed for read — log_buffer appends are under lock in timer; read here is best-effort)
      - If `job is None: return`
      - If `job.state in TERMINAL: yield from job.log_buffer; return`
      - While non-terminal: yield nothing (SLURM log-on-completion model per D-58); caller uses `wait_for_state` before calling `log_iter`
    - `_persist_state(self) -> None`:
      - If `self._state_file is None: return`
      - Under `_lock`: data = `[j._to_json() for j in self._jobs.values()]`
      - Use `_atomic_write_text` (import from `automil.cli.lifecycle._shared`) to write JSON array
    - `_load_state(self) -> None`:
      - Read `self._state_file` as JSON array; for each item: `job = _MockJob._from_json(item)`; `self._jobs[job.handle.opaque_id] = job`

## Threat Model (STRIDE + disposition)

| ID | Threat | STRIDE | Severity | Disposition |
|----|--------|--------|----------|-------------|
| T-02-06-S01 | `threading.Timer` callback deadlocks by trying to acquire `_lock` while main thread holds it | Denial of Service | HIGH | Mitigate: Timer callbacks ONLY set state under `_lock` and exit. They NEVER call `poll()`, `cancel()`, or any other Backend method. Verified by code review: `_finish` acquires `_lock` once, writes `job.state`, exits. No nested lock acquisition. (RESEARCH.md §6 Pitfall 3) |
| T-02-06-S02 | `cancel_requested` flag read as a bare `bool` causes race condition | Tampering | HIGH | Mitigate: `cancel_requested` is `threading.Event` — `.set()` is atomic across threads; `.is_set()` reads the atomic flag. Never use a bare `bool` field (RESEARCH.md §6 / Don't Hand-Roll table) |
| T-02-06-S03 | Daemon timer thread keeps running after test process exits, causing test hangs | Denial of Service | MEDIUM | Mitigate: `t.daemon = True` on all `threading.Timer` instances — daemon threads are killed automatically when the main process exits (Python threading docs) |
| T-02-06-S04 | `mock_slurm` accidentally importable from `automil.backends` namespace at production config time | Elevation of Privilege | MEDIUM | Mitigate: D-69 — not imported in `__init__.py`; verified by T-02-06-02 grep gate |
| T-02-06-S05 | `threading.Event` not JSON-serialisable; `_to_json` accidentally tries to include it | Information Disclosure | LOW | Mitigate: `_to_json` explicitly lists serialised fields (node_id, backend, opaque_id, submitted_at, state, log_buffer) and does NOT include `cancel_requested` or `timer` |
