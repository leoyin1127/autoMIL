# Plan 02-05: `LocalBackend` — Thin Protocol Adapter over `_orchestrator_daemon`

## Objective

Implement `LocalBackend(Backend)` in `src/automil/backends/local.py` as a thin protocol adapter. Each of the five methods maps to the existing daemon's on-disk state machine — no new logic, no new process control, no new file formats. Per D-60, the 387-test suite is the correctness gate: if all 387 pass after this plan, LocalBackend is behaviourally identical to the pre-Phase-2 orchestrator.

Per D-61, `__init__` takes `(project_root, automil_dir)` and stores `self._daemon = ExperimentOrchestrator(...)`. Per D-77, `submit` writes to `queue/` and returns `JobHandle(opaque_id="pending")` — the daemon updates `opaque_id` to the real PID when it launches. Per BCK-04, `local.py` is the ONLY non-daemon file allowed to use `os.kill` / `Popen` / `.pid` references — but it should delegate these to `_daemon` methods rather than calling them directly.

After this plan, `backends/__init__.py` is updated to activate the auto-import so `BACKENDS["local"]` is populated at package import time.

Purpose: Without `LocalBackend`, the `cancel` and `resubmit` CLI commands (Plan 02-08) cannot dispatch to local jobs.

Output: `local.py` (~120 lines), minor update to `__init__.py` (activate the TODO placeholder). All 387 tests green.

  - `submit(self, spec: JobSpec) -> JobHandle`:
    - Build the queue spec dict from `spec` fields (map `JobSpec` -> the existing `queue/<id>.json` JSON shape that `_orchestrator_daemon` expects)
    - Write to `self._daemon._automil_dir / "orchestrator" / "queue" / f"{spec.node_id}.json"` using `_atomic_write_text` (from `automil.cli.lifecycle._shared` — import inside method to avoid circular)
    - Return `JobHandle(node_id=spec.node_id, backend="local", opaque_id="pending", submitted_at=time.time())`
    - Note: `opaque_id="pending"` because daemon hasn't launched yet (D-77)
  - `poll(self, handle: JobHandle) -> JobState`:
    - Check `running/<handle.node_id>.json` — if exists → `RUNNING`
    - Check `archive/<handle.node_id>/result.json` — if exists, read `status` field → map to `JobState` (completed, crashed, cancelled, etc.)
    - Check `queue/<handle.node_id>.json` — if exists → `PENDING`
    - Otherwise → `PENDING` (not yet picked up)
    - Return pure snapshot, no side effects
  - `cancel(self, handle: JobHandle, signal: Optional[int] = None) -> None`:
    - Read `running/<handle.node_id>.json` to get the PID
    - Call `self._daemon._kill_experiment(handle.node_id)` (or the equivalent method — match the actual method name from the read pass)
    - Fire-and-forget: return None immediately (per D-57)
    - Log a warning if `signal` is not None: "custom signal ignored by LocalBackend via _kill_experiment; use standard cancel"
  - `list_running(self) -> list[JobHandle]`:
    - Scan `self._daemon._automil_dir / "orchestrator" / "running" / "*.json"`
    - For each file, read `node_id` (from filename or JSON content), construct `JobHandle(node_id=..., backend="local", opaque_id=str(pid_from_file), submitted_at=mtime_or_field)`
    - Return list of handles — restart-safe (D-59)
  - `log_iter(self, handle: JobHandle) -> Iterator[str]`:
    - Find the log file at `archive/<handle.node_id>/run.log` or equivalent (check from read pass)
    - If job is in terminal state, yield the buffered file content and stop
    - If job is running, tail with 0.1s tick (D-58): `while not terminal: yield new_lines; time.sleep(0.1)`
    - Close iterator when job reaches terminal state

## Threat Model (STRIDE + disposition)

| ID | Threat | STRIDE | Severity | Disposition |
|----|--------|--------|----------|-------------|
| T-02-05-S01 | `_recover_orphans()` triggered via LocalBackend construction from CLI commands | Tampering | HIGH | Mitigate: `LocalBackend.__init__` stores daemon with `recover=False` (existing ExperimentOrchestrator constructor already does this — verify in read pass); add assertion in docstring and inline comment; test: call `LocalBackend(...)` from a CLI-like context with a running job and verify job is NOT marked crashed |
| T-02-05-S02 | `cancel()` sends signal to wrong PID (PID reuse between submit and cancel) | Spoofing | MEDIUM | Mitigate: `_kill_experiment` in `_orchestrator_daemon.py` already cross-checks process start time via `/proc/<pid>/stat` (CLN-04 fix from Phase 0); `local.py` delegates to it rather than calling `os.kill` directly |
| T-02-05-S03 | `log_iter` hangs forever if terminal state never observed | Denial of Service | MEDIUM | Mitigate: contract test (`test_log_iter_terminal_close`) must pass against LocalBackend; `log_iter` must check terminal state on each tick and break out; 0.1s tick ensures CI-friendly behavior |
| T-02-05-S04 | `submit()` writes to `queue/` while daemon is not running — spec file never picked up | Denial of Service | LOW | Accept: this is expected behaviour (existing queue-file model); the caller is responsible for starting the daemon; `LocalBackend.poll()` returns PENDING indefinitely for unprocessed queue entries |
