# Plan 02-02: BACKENDS Registry Singleton + `register` Decorator + BackendError

## Objective

Ship the backend registry infrastructure: `BACKENDS` dict, `register(name)` class decorator, and `BackendError`. This is an exact analog of Phase 1's `registrar.py` + `_state.py` split — the key difference is that Phase 2's registry is simpler (one kind, string keys) so it lives directly in `backends/__init__.py` rather than a separate `_state.py`. Plans 02-05 and 02-06 use `@register("local")` and `@register("mock_slurm")` respectively; Plan 02-08's `cancel.py` uses `BACKENDS[name]` to dispatch to the right backend at runtime.

Per D-68, `BACKENDS` is populated via decorator — the same pattern as Phase 1's `MODEL_VARIANTS`. Per D-69, `mock_slurm` is NOT auto-imported in `__init__.py` — tests do it explicitly. `BackendError` is a lightweight exception subclass used for registration errors.

Purpose: Without the registry, CLI commands cannot resolve backend names at runtime. Without `BackendError`, registration errors are generic `TypeError`.

    ```
    module docstring referencing BCK-01, D-51, D-68, D-69
    from __future__ import annotations
    import logging
    from typing import TYPE_CHECKING

    # Re-exports from base.py (installed by Plan 02-01)
    from automil.backends.base import Backend, BackendError (NO — BackendError is from errors.py)
    from automil.backends.base import Backend, JobHandle, JobSpec, JobState
    from automil.backends.errors import BackendError

    logger = logging.getLogger(__name__)

    # Registry (D-68)
    BACKENDS: dict[str, type[Backend]] = {}

    def register(name: str):
        """Class decorator: register a Backend subclass under `name`.

        Usage: @register("local")
               class LocalBackend(Backend): ...

        D-68: mock_slurm is NOT auto-imported here (D-69); tests import it explicitly.
        """
        def _decorator(cls: type[Backend]) -> type[Backend]:
            if not (isinstance(cls, type) and issubclass(cls, Backend)):
                raise BackendError(
                    f"{cls.__name__} must subclass Backend to be registered"
                )
            if name in BACKENDS:
                raise BackendError(
                    f"Backend {name!r} is already registered as {BACKENDS[name].__name__}. "
                    f"Duplicate registration rejected."
                )
            BACKENDS[name] = cls
            logger.info("Registered backend %r -> %s", name, cls.__name__)
            return cls
        return _decorator

    def _clear_backends() -> None:
        """Test-only: clear BACKENDS registry for isolation. Never call in production."""
        BACKENDS.clear()

    __all__ = [
        "Backend", "JobHandle", "JobSpec", "JobState",
        "BackendError", "BACKENDS", "register", "_clear_backends",
    ]
    ```
