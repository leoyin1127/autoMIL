# Requirements: autoMIL — F2-readiness framework refactor

**Defined:** 2026-05-01
**Core Value:** An agent can autonomously discover model improvements for any user's training code under a 6-hour-per-cell budget, with discovered variants reproducible, attributable to their parents, and portable across machines and LLM runtimes.

## v1 Requirements

Scope of this milestone. Each maps to exactly one roadmap phase (filled in during Step 8). REQ-IDs grouped by component category.

### Cleanup (CLN)

<!-- Tier-2 hygiene + CLI restructuring; precondition for new commands to land cleanly. -->

- [ ] **CLN-01**: Root `.gitignore` excludes all CONCERNS-flagged HIGH-severity runtime artefacts (Tier 2 follow-ups beyond the Tier 1 .automil_worktrees/.automil_active fixes already shipped)
- [ ] **CLN-02**: Subprocess `env` whitelisted (no full `os.environ` inheritance to experiment processes); `AUTOBENCH_*` and explicit allowlist only
- [ ] **CLN-03**: `.env` parser replaced with `python-dotenv`; quoted values, `export ` prefix, comments after `=` correctly handled
- [ ] **CLN-04**: PID-file stale-detection cross-checks process start time (mitigates PID reuse race)
- [ ] **CLN-05**: `nvidia-smi` invocation pins the binary path or reports detection method, not bare `$PATH` lookup
- [ ] **CLN-06**: `cli.py` (726 lines) split into per-command-group files under `src/automil/cli/` with a thin `__init__.py` aggregator; existing `automil <subcommand>` invocations unchanged
- [ ] **CLN-07**: `compat.py` re-export shim preserves all current `from automil.X import Y` import paths; deprecation table documented

### Registry (REG)

<!-- The keystone. Variants as committed code modules; shared library files become read-only. -->

- [ ] **REG-01**: `Variant` ABC defined with `forward(features, coords) -> AggregatorOutput`-style stable interface; `VariantSpec` frozen dataclass captures `parent`, `base_commit`, `mutations`, `composite`, `kind` (architectural | recipe | training-policy | inference)
- [ ] **REG-02**: Internal `Registry` class with `@register` decorator + `importlib.metadata.entry_points` cross-project discovery; per-parent `variants/__init__.py` regenerated by `automil refresh-registry` (CLI, deterministic ordering)
- [ ] **REG-03**: Validator chain (`identity`, `interface`, `purity`) runs as a pre-submit hook; identity-preservation rule pre-registered in `automil/config.yaml` (`registry.identity_constraints`)
- [ ] **REG-04**: `registry.protected` config field lists files the agent's overlay MUST NOT touch (e.g., `benchmarks/lib/CLAM/`, `models/clam/base.py`); `automil submit` rejects overlays touching protected paths
- [ ] **REG-05**: `automil check` fails on uncommitted edits to protected files
- [ ] **REG-06**: Search-scope mode flag (`architecture-preserving` | `free`) selects which validator chain runs; default `free`; opt-in `architecture-preserving` per phase config
- [ ] **REG-07**: `train.py` reads model class, loss class, training policies, hyperparameters from `automil/config.yaml`; zero `args.X = literal` overrides remain in framework code
- [ ] **REG-08**: CCRCC current dirty edits ported to registered variant modules under `experiments/ccrcc/automil/variants/clam_mb/clam_mb_v0176.py` + `losses/variants/ce_smooth008.py` + `training/policies/sam_lookahead.py`; manifest with parent commit + composite + node id committed
- [ ] **REG-09**: Reproduction sanity check passes — CCRCC `node_0176` reproduces composite within ±0.005 from a clean checkout via the registry path (no shared-file edits in working tree)

### Backend (BCK)

<!-- Pluggable orchestrator: same code on laptop, SLURM cluster, Ray cluster. -->

- [x] **BCK-01**: `Backend` ABC with `submit(spec) -> JobHandle`, `poll(handle) -> JobState`, `list_running() -> [JobHandle]`, `cancel(handle, signal) -> None`, `log_iter(handle) -> Iterator[str]`; state-not-control-flow `JobState` enum (`pending | running | completed | crashed | cancelled | budget_killed`)
- [x] **BCK-02**: `LocalBackend` ships as a re-export shim over the existing 750-line orchestrator; behavioural diff vs current orchestrator = empty (verified by existing 48-test suite passing)
- [x] **BCK-03**: `MockSLURMBackend` test fixture: simulates eventual-consistency status (5s poll lag), opaque `job_id`, fire-and-forget `cancel`, node-local filesystem; used to validate the ABC against ≥2 implementations BEFORE locking
- [x] **BCK-04**: Lint check (ruff/mypy custom rule) forbids `os.kill`, `Popen`, `pid` references outside `backends/local.py` and `backends/_orchestrator_daemon.py`
- [x] **BCK-05**: `SLURMBackend` (`backends/slurm.py`) on top of `submitit>=1.5.3`; opt-in via `pip install -e '.[slurm]'`; `--time --signal=B:TERM@30` SLURM directives match framework's wall-clock contract
- [x] **BCK-06**: `RayBackend` (`backends/ray.py`) on top of `ray>=2.55.1` raw `ray.remote` + placement groups (NOT `ray.tune`); opt-in via `pip install -e '.[ray]'`; `ray.get(timeout=...)` + `ray.cancel(force=True)` honour wall-clock contract

### Trajectory (TRJ)

<!-- Reproducibility instrumentation: snapshot agent prompts + tool-call traces per submit. -->

- [ ] **TRJ-01**: `archive/<node_id>/trajectory.jsonl` is the canonical artifact; one event per line; first line is metadata `{schema_version, runtime, runtime_version, tool_schema_version, automil_version, automil_runtime_env}`
- [ ] **TRJ-02**: Field names follow OpenTelemetry `gen_ai.*` semantic conventions (e.g., `gen_ai.system`, `gen_ai.request.model`, `gen_ai.tool.name`, `gen_ai.event.name`); no `opentelemetry-sdk` runtime dependency
- [ ] **TRJ-03**: Redaction-on-capture (regex matchers for `sk-…`, `hf_…`, `ghp_…`, AWS access keys, `*_API_KEY=…`, `*_TOKEN=…`); per-event 8 KB cap; per-file 5 MB soft / 50 MB hard rotate; rotation produces `trajectory.<n>.jsonl` siblings
- [ ] **TRJ-04**: Trajectory recorder integrated as a runtime hook for ≥2 supported runtimes (Claude Code, opencode); CLI fallback `automil trajectory record <event-json>` covers runtimes without native hooks
- [ ] **TRJ-05**: `archive/<node_id>/trajectory.jsonl` gitignored by default; explicit `automil trajectory export` produces a redacted, schema-validated bundle for sharing
- [ ] **TRJ-06**: Tests cover schema-version mismatch tolerance and redaction-pattern coverage (positive cases for each leak class above)

### Multi-runtime (MRT)

<!-- Framework runs identically across LLM agent runtimes. Defends F2's "Claude paper" reviewer attack. -->

- [ ] **MRT-01**: `src/automil/agent_assets/` reorganised: `_shared/SKILL.md` is canonical content; per-runtime subdirectories (`claude/`, `codex/`, `opencode/`) contain ONLY diffs/overrides, not full duplicates
- [ ] **MRT-02**: `AGENTS.md` is generated/installed at the project root by `automil init`; contains the autoMIL-specific instructions universal to all runtimes
- [ ] **MRT-03**: `automil init` accepts `--runtime <claude|codex|opencode|deepseek-via-X>` and detects existing `.claude/`, `.codex/`, `.opencode/` directories; installs the right runtime's overlay assets
- [ ] **MRT-04**: `automil show-skill --runtime <name>` debug command renders the final per-runtime SKILL/AGENTS file as it would be installed
- [ ] **MRT-05**: End-to-end smoke test passes for ≥2 runtimes (Claude Code + one of {opencode, codex}): an experiment loop submits, runs, completes, and writes a valid `result.json`
- [ ] **MRT-06**: `agent_assets/deepseek/README.md` documents that DeepSeek is a *model*, not a runtime — routed through opencode/Codex/etc.

### Cap (CAP)

<!-- 6h hard wall-clock per cell, framework-enforced; partial-fold protocol prevents corruption. -->

- [ ] **CAP-01**: `(dataset, encoder, parent_id)` cell is a first-class graph entity with `cell_id`; `cells/<cell_id>.json` state file persists `started_at`, `budget_seconds`, `consumed_seconds`, status (`active | refusing-new | terminating | finalized`)
- [ ] **CAP-02**: Two-tier cap: at `T - safety_buffer` (default 30 min) the cell enters `refusing-new` (orchestrator rejects new submits for this cell); at `T` cell enters `terminating` (running experiments killed via `Backend.cancel`)
- [ ] **CAP-03**: Per-fold checkpoint protocol: `train.py` writes `fold_<i>_result.json` after each fold; `result.json` is the aggregate across completed folds (`status: partial | completed`)
- [ ] **CAP-04**: Budget-killed experiments reconcile to `executed` (NOT `crash`) with whatever partial composite is computable from completed folds; descendant cascade recomputes against the partial composite, not against zero
- [ ] **CAP-05**: `cell_started_at` and `consumed_seconds` survive daemon restart (persisted in `cells/<cell_id>.json`, not in-memory only)
- [ ] **CAP-06**: `automil cell status [cell_id]` and `automil cell list` CLI surfaces budget state for operator inspection

### Gate (GTE)

<!-- Generalization gate: candidate variants must improve on ≥K held-out cells before promotion. -->

- [ ] **GTE-01**: New node status `candidate` (between `executed/keep` and `registered`); `candidate` nodes have a `gate_manifest.json` listing held-out cells they must improve on
- [ ] **GTE-02**: Held-out cells declared per `gate_manifest.json` BEFORE the search starts (pre-registration); manifest committed to git with cell ids and statistical-test parameters
- [ ] **GTE-03**: Gate spawns held-out evaluations via `Backend.submit()` (same path as agent submits, NOT a parallel mechanism); gate node has child eval-nodes with explicit `gate_eval` edge type
- [ ] **GTE-04**: Promotion criterion: paired Wilcoxon (or comparable test from `gate_manifest.json`) with bootstrap CI (1000 reps) and Bonferroni correction across held-out cells; `K` (minimum cells passed) and `p_threshold` are config-set
- [ ] **GTE-05**: Manual nomination is the default (`auto_nominate: false`); the agent or operator explicitly nominates a node via `automil nominate <node>`
- [ ] **GTE-06**: Promotion-rate metric exposed in viz dashboard and `automil status` (% of nominated candidates that passed gate); used as a search-health signal

### CLI (CLI)

<!-- Lifecycle commands the agent triggers during normal loop operation. -->

- [ ] **CLI-01**: `automil apply <node_id>` copies a node's variant module + config delta back to the working tree
- [ ] **CLI-02**: `automil revert-baseline` resets all `registry.editable` paths to their `base_commit` state (idempotent)
- [x] **CLI-03**: `automil cancel <node_id>` terminates a running experiment via the backend's `cancel`; archives current state with `status: cancelled`
- [x] **CLI-04**: `automil resubmit <node_id>` re-queues a crashed/cancelled node with a fresh worktree
- [ ] **CLI-05**: `automil port-variant <node_id>` converts a node's overlay (dirty edits) into a registered variant module + manifest; idempotent; rejects nodes that are already registered
- [ ] **CLI-06**: `automil promote-variant <node_id>` moves a candidate variant into the canonical `variants/` directory, writes provenance manifest, and stages for commit
- [ ] **CLI-07**: `automil reconcile --recompute-best` rebuilds `meta.best_node_id` from the honest non-leaky composite walking only `executed/keep` nodes (closes T2 from earlier this session)
- [ ] **CLI-08**: `automil refresh-registry` regenerates per-parent `variants/__init__.py` files from disk; deterministic ordering; idempotent
- [ ] **CLI-09**: `automil check` validates project setup: protected-files clean, registry consistency, env vars declared, runtime asset present, sample-size sanity

### Setup (STP)

<!-- One-shot bootstrap skill; hardware autodetection feeds setup defaults. -->

- [ ] **STP-01**: `LocalBackend.healthcheck()` reports detected GPU count, VRAM per GPU, accelerator type (CUDA / ROCm / CPU), Python version, autoMIL version
- [ ] **STP-02**: `automil init` consumes healthcheck output and pre-fills `automil/config.yaml` defaults (max_concurrent_per_gpu, default_vram_estimate_gb derived from quantile_95 of empirical `results.tsv` if available, else conservative defaults)
- [ ] **STP-03**: Hardware-detect produces a *report*, not a *decision*: if detection fails, `automil init` prints the detected values and prompts override; never silently uses wrong defaults
- [ ] **STP-04**: `/automil-setup` skill (Claude format `.claude/skills/automil-setup/SKILL.md`) inspects an arbitrary user repo, identifies the training entry point, drafts `automil/config.yaml` + `program.md`, scaffolds a starter `variants/` skeleton, picks defaults from healthcheck
- [ ] **STP-05**: `/automil-setup` skill is idempotent — re-running on an already-initialised project diffs and updates rather than overwriting
- [ ] **STP-06**: After setup, mandatory `automil check` + 1-minute dry-run experiment; setup is not "done" until both pass
- [ ] **STP-07**: Skill ships per-runtime overlays: `_shared/automil-setup/SKILL.md` canonical, `claude/automil-setup/`, `codex/automil-setup/`, `opencode/automil-setup/` overrides

### Decouple (DEC)

<!-- Framework genuinely consumer-agnostic — autobench is one of many possible users. -->

- [ ] **DEC-01**: `grep -r "autobench\|AUTOBENCH_" src/automil/` returns zero matches; equivalent for any other autobench-specific identifier
- [ ] **DEC-02**: A second consumer (sklearn-iris classifier as a minimal training script) plugs into autoMIL via documented contract and runs an experiment loop end-to-end
- [ ] **DEC-03**: `result.json` schema is JSON-Schema-validated at ingestion; orchestrator rejects malformed results with a clear error pointing to the schema location
- [ ] **DEC-04**: Composite scoring formula is config-driven (`automil/config.yaml: scoring.formula` or `scoring.entry_point`); not hardcoded to autobench's 4-key (val_auc + val_bacc + test_auc + test_bacc) recipe
- [ ] **DEC-05**: Required env vars are declared in `automil/config.yaml: env.required` and validated by `automil check`; missing vars fail fast at startup, not deep inside training code
- [ ] **DEC-06**: `docs/training-script-contract.md` documents what a training script must do to be autoMIL-compatible: write `result.json`, accept `CUDA_VISIBLE_DEVICES`, exit cleanly on SIGTERM with partial-fold output, declared env vars
- [ ] **DEC-07**: Final reproduction sanity (mirrors REG-09): clean checkout, registry path, fresh worktree, CCRCC `node_0176` reproduces ±0.005 — and the same harness runs the sklearn-iris consumer end-to-end as the decoupling proof

### F2 Experimental Grid (F2)

- **F2-01**: Per-parent search runs across 5 parent MIL models (CLAM_MB, AB-MIL, DSMIL, TransMIL, DTFD-MIL)
- **F2-02**: Standardized 18-cell evaluation grid (6 datasets × 3 encoders) with paired Wilcoxon + Bonferroni
- **F2-03**: Mutation-dimension ablation (architectural, novel loss, novel training paradigm, novel inference, aggressive recipe)
- **F2-04**: Released `Model-autoMIL` PyTorch modules + recipe.yaml per parent
- **F2-05**: Public leaderboard (Patho-Bench-compatible) hosting

- **CMR-02**: Framework-level cross-runtime composite (which runtime found which mutation)

### Other deferred

- **MAC-01**: macOS / MPS support
- **STAR-01**: Anti-starvation aging in scheduler
- **COST-01**: Per-trial $-cost tracking

- **PERF-01**: O(N²) → O(N) `recalculate_scores` (defer until graphs >1k nodes)

## Out of Scope

Explicitly excluded from v1 *and* v2 unless reopened with discussion.

| Feature | Reason |
|---|---|
| Full F2 paper experimental grid (5 parents × 18 cells × 4 recipes × 25 measurements) | Future milestone — framework is the precondition |
| Paper writing and venue submission | Future milestone — not engineering scope |
| F1 as originally proposed (architecture-preserving recipe-only standardization paper) | Denied; subsumed into F2-style architectural exploration with a `--mode=architecture-preserving` flag |
| Mac / MPS / non-CUDA accelerators | Linux + CUDA only for v1; revisit on real user pull |
| Replacing `/automil-setup` skill with a CLI subcommand | Skill is correct vehicle: one-shot, autonomous, context-heavy; CLI would force re-invocation each session and burn context |
| Multi-LLM concurrent orchestration on the same problem | Framework-level multi-runtime *support* is in v1; concurrent same-problem orchestration is paper-time ablation |
| Containerized execution sandboxing | Security-relevant lift; documented as a known boundary, not a v1 feature |
